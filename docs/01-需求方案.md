# Jenkins API Tool - 需求方案文档

## 1. 项目概述

### 1.1 项目背景
开发一个基于 Node.js 的 Jenkins RESTful API 客户端工具,完成 Jenkins 构建操作的核心 HTTP 功能,包括带参数触发构建、查询构建状态、下载构建产物。

### 1.2 技术栈
- **运行时**: Node.js (>= 18.0.0)
- **开发语言**: TypeScript
- **HTTP 客户端**: axios
- **配置文件**: dotenv (支持 .env 文件)

### 1.3 项目约束
- 仅使用 Node.js 生态,不涉及 Go、Java 等后端语言

### 1.4 迭代计划

| 期数 | 范围 | 目标 |
|------|------|------|
| 第一期 | HTTP 核心功能 | 完成 Jenkins API 调用核心逻辑,提供可编程的 SDK 接口 |
| 第二期 | CLI 命令化 | 基于第一期 HTTP 功能,封装命令行工具 |

**本期(第一期)专注于 HTTP 核心功能实现和验证。**

## 2. 功能需求 (第一期)

### 2.1 核心功能

#### 2.1.1 带参数触发构建 (build)
- 支持指定 Jenkins Job 名称触发构建
- 支持多种参数类型:
  - 字符串参数 (String)
  - 选项参数 (Choice)
  - 布尔参数 (Boolean)
  - 文件参数 (File) - 支持文件上传
- 支持异步触发 (不等待构建完成)
- 支持同步触发 (等待构建完成并返回结果)
- 返回构建队列 ID 或构建编号

#### 2.1.2 查询构建状态 (status)
- 根据 Job 名称和构建编号查询构建状态
- 支持查询最后一次构建状态
- 返回信息包括:
  - 构建状态 (SUCCESS, FAILURE, ABORTED, IN_PROGRESS 等)
  - 构建编号
  - 构建开始/结束时间
  - 构建日志 URL
  - 构建产物列表

#### 2.1.3 下载构建产物 (download)
- 根据 Job 名称和构建编号下载指定产物
- 支持下载所有产物
- 支持指定本地下载目录
- 返回下载结果信息

### 2.2 辅助功能

#### 2.2.1 配置管理
- 支持通过 `.env` 文件或代码配置 Jenkins 连接信息
- 支持多实例配置切换

#### 2.2.2 日志输出
- 支持不同日志级别 (info, debug, error)
- 控制台格式化输出

## 3. Jenkins API 对接

### 3.1 认证方式
- **用户名密码认证**: 使用用户名 + 密码进行 Basic Auth
  - 请求头格式: `Authorization: Basic base64(username:password)`
- **API Token 认证**: 使用用户名 + API Token 进行 Basic Auth
  - 请求头格式: `Authorization: Basic base64(username:apiToken)`
- **自动选择**: 优先使用 Token,如果未配置则使用密码,两者都未配置则报错

### 3.2 核心 API 端点

| 功能 | 方法 | 端点 |
|------|------|------|
| 触发无参构建 | POST | `/job/{jobName}/build` |
| 触发带参构建 | POST | `/job/{jobName}/buildWithParameters` |
| 获取构建信息 | GET | `/job/{jobName}/{buildNumber}/api/json` |
| 获取构建日志 | GET | `/job/{jobName}/{buildNumber}/consoleText` |
| 获取最后一次构建 | GET | `/job/{jobName}/lastBuild/api/json` |
| 下载构建产物 | GET | `/job/{jobName}/{buildNumber}/artifact/{relativePath}` |
| 获取队列信息 | GET | `/queue/item/{queueId}/api/json` |

### 3.3 构建参数传递格式

#### JSON 格式 (推荐)
```json
{
  "parameter": [
    {"name": "param1", "value": "value1"},
    {"name": "param2", "value": "true"},
    {"name": "file1", "file": "fileContent"}
  ]
}
```

#### Form Data 格式
```
Content-Type: multipart/form-data
param1=value1&param2=true
```

## 4. SDK 接口设计 (第一期)

### 4.1 初始化配置
```typescript
import { JenkinsClient } from './src';

const client = new JenkinsClient({
  url: 'http://jenkins.example.com',
  username: 'your_username',
  apiToken: 'your_api_token'
});
```

### 4.2 触发构建接口
```typescript
// 异步触发
const result = await client.build('my-job', {
  ENV: 'prod',
  VERSION: '1.0.0'
});

// 同步触发 (等待完成)
const result = await client.build('my-job', {
  ENV: 'prod'
}, {
  wait: true,
  pollInterval: 5000  // 轮询间隔(ms)
});

// 带文件参数
const result = await client.build('my-job', {
  CONFIG: { type: 'file', path: './config.json' }
});
```

### 4.3 查询构建状态接口
```typescript
// 查询指定构建
const status = await client.getStatus('my-job', 123);

// 查询最后一次构建
const status = await client.getStatus('my-job', 'last');
```

### 4.4 下载构建产物接口
```typescript
// 下载单个产物
await client.download('my-job', 123, 'target/app.jar', './dist');

// 下载所有产物
await client.downloadAll('my-job', 123, './dist');
```

## 5. 配置设计

### 5.1 .env 文件格式
```env
# Jenkins 服务器地址
JENKINS_URL=http://localhost:8080

# 认证信息 (二选一,推荐使用 API Token)
# 方式1: API Token 认证
JENKINS_USERNAME=your_username
JENKINS_API_TOKEN=your_api_token

# 方式2: 用户名密码认证
# JENKINS_USERNAME=your_username
# JENKINS_PASSWORD=your_password
```

### 5.2 代码配置覆盖
```typescript
const client = new JenkinsClient({
  url: process.env.JENKINS_URL,
  username: process.env.JENKINS_USERNAME,
  apiToken: process.env.JENKINS_API_TOKEN
});
```

## 7. 错误处理

### 7.1 错误类型
- 网络连接错误
- 认证失败 (401/403)
- Job 不存在 (404)
- 构建仍在进行中
- 产物不存在

### 7.2 错误输出
```
✗ 错误: 构建失败
  Job: my-job
  构建编号: #123
  状态: FAILURE
  日志: http://jenkins.example.com/job/my-job/123/console
```
