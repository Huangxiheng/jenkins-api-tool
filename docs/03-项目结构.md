# Jenkins API Tool - 项目结构文档

## 1. 项目目录结构

```
jenkins-api-tool/
│
├── src/                              # 源代码目录
│   ├── index.ts                      # SDK 入口文件 (统一导出)
│   │
│   ├── client/                       # 客户端层
│   │   └── jenkins-client.ts         # JenkinsClient 主客户端类
│   │
│   ├── services/                     # 服务层 (核心业务逻辑)
│   │   ├── http-client.ts            # HTTP 客户端封装
│   │   ├── build-service.ts          # 构建服务
│   │   ├── status-service.ts         # 状态查询服务
│   │   └── download-service.ts       # 下载服务
│   │
│   ├── types/                        # 类型定义层
│   │   └── index.ts                  # 所有 TypeScript 类型和接口
│   │
│   ├── errors/                       # 错误处理层

│   │
│   ├── utils/                        # 工具层

```

│                        使用者 (SDK Consumer)                     │

                             │
┌────────────────────────────▼────────────────────────────────────┐
│                    支撑层 (types/errors/utils/config)            │
│  ┌──────────┐ ┌────────┐ ┌─────────────┐ ┌────────┐            │
│  │  Types   │ │ Errors │ │ Logger/Utils│ │ Config │            │

### 3.2 客户端模块 - `src/client/jenkins-client.ts`

| 属性 | 说明 |
|------|------|
| 路径 | `src/client/jenkins-client.ts` |
| 类名 | `JenkinsClient` |
| 职责 | SDK 主入口类,组合所有服务,对外提供统一 API |
| 依赖 | `HttpClient`, `BuildService`, `StatusService`, `DownloadService` |

**核心方法**:

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `build(jobName, params?, options?)` | Job名称, 构建参数(可选), 构建选项(可选) | `BuildTriggerResult` \| `BuildCompleteResult` | 触发构建。异步返回队列ID,同步( options.wait=true )返回完整构建结果 |
| `getStatus(jobName, buildNumber)` | Job名称, 构建编号或 'last' | `BuildStatusResult` | 查询指定构建的状态信息 |
| `download(jobName, buildNumber, artifactPath, outputDir)` | Job名称, 构建编号, 产物路径, 输出目录 | `DownloadResult` | 下载单个构建产物 |
| `downloadAll(jobName, buildNumber, outputDir)` | Job名称, 构建编号, 输出目录 | `DownloadAllResult` | 下载该构建的所有产物 |
| `getConsoleText(jobName, buildNumber)` | Job名称, 构建编号 | `string` | 获取构建控制台日志文本 |
| `verifyAuth()` | 无 | `{ authenticated, user?, version?, url }` | 验证 Jenkins 连接和认证是否有效 |

**内部属性**:

| 属性 | 类型 | 说明 |
|------|------|------|
| `config` | `Required<JenkinsClientConfig>` | 合并默认值后的完整配置 |
| `httpClient` | `HttpClient` | HTTP 客户端实例 |
| `buildService` | `BuildService` | 构建服务实例 |
| `statusService` | `StatusService` | 状态查询服务实例 |
| `downloadService` | `DownloadService` | 下载服务实例 |
| `logger` | `Logger` | 日志记录器 |

### 3.3 HTTP 客户端模块 - `src/services/http-client.ts`

| 属性 | 说明 |
|------|------|
| 路径 | `src/services/http-client.ts` |
| 类名 | `HttpClient` |
| 职责 | 封装 axios HTTP 请求,处理认证、CSRF、错误、文件下载 |
| 依赖 | `axios`, `fs`, `path` |

**核心方法**:

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `getBaseUrl()` | 无 | `string` | 获取 Jenkins 服务器基础 URL |
| `initCrumb()` | 无 | `Promise<void>` | 获取 CSRF crumb token,用于 CSRF 保护 |
| `get<T>(urlPath, params?)` | 请求路径, 查询参数(可选) | `Promise<T>` | 发送 GET 请求 |
| `post<T>(urlPath, data?, headers?)` | 请求路径, 请求体(可选), 请求头(可选) | `Promise<{data, headers}>` | 发送 POST 请求,返回数据和响应头 |
| `download(urlPath, outputPath, onProgress?)` | 请求路径, 输出路径, 进度回调(可选) | `Promise<void>` | 流式下载文件,支持进度回调 |
| `handleAxiosError(error)` | Axios 错误对象 | `never` (抛出异常) | 统一错误处理,转换为自定义错误类型 |

**错误映射**:

| HTTP 状态码 | 抛出的错误类型 |
|-------------|---------------|
| 401 / 403 | `AuthenticationError` |
| 404 | `JobNotFoundError` / `JenkinsError` |
| 超时 | `NetworkError` |
| 其他 | `JenkinsError` |

### 3.4 构建服务模块 - `src/services/build-service.ts`

| 属性 | 说明 |
|------|------|
| 路径 | `src/services/build-service.ts` |
| 类名 | `BuildService` |
| 职责 | 处理 Jenkins 构建触发和完成等待逻辑 |
| 依赖 | `HttpClient`, `StatusService`, `form-data`, `fs` |

**核心方法**:

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `trigger(jobName, params?, options?)` | Job名称, 构建参数(可选), 构建选项(可选) | `Promise<BuildTriggerResult>` | 触发构建,返回队列 ID |
| `waitForCompletion(jobName, queueId, options)` | Job名称, 队列ID, 轮询选项 | `Promise<BuildCompleteResult>` | 轮询等待构建完成,返回完整结果 |
| `triggerWithoutParameters(jobName)` | Job名称 | `Promise<number>` | 触发无参数构建 (内部方法) |
| `triggerWithParameters(jobName, params)` | Job名称, 构建参数 | `Promise<number>` | 触发带参数构建 (内部方法) |
| `prepareFormData(params)` | 构建参数 | `Promise<FormData>` | 准备 FormData (处理文件上传) |
| `prepareQueryParams(params)` | 构建参数 | `Record<string, string>` | 准备 URL 查询参数 |
| `extractQueueId(response)` | POST 响应 | `number` | 从响应头 Location 中提取队列 ID |
| `handleBuildTriggerError(error, jobName)` | 错误对象, Job名称 | `never` | 处理构建触发错误 |
| `sleep(ms)` | 延迟毫秒数 | `Promise<void>` | 延迟函数 |

**构建触发策略**:

| 场景 | 请求方式 | 端点 |
|------|----------|------|
| 无参数构建 | POST (application/x-www-form-urlencoded) | `/job/{jobName}/build` |
| 简单参数 (String/Boolean/Choice) | POST (query string) | `/job/{jobName}/buildWithParameters?KEY=VALUE` |
| 含文件参数 | POST (multipart/form-data) | `/job/{jobName}/buildWithParameters` |

**轮询等待逻辑**:

```
1. 轮询队列状态 → GET /queue/item/{queueId}/api/json
   └─ 检查 executable 字段,获取构建编号
2. 轮询构建状态 → GET /job/{jobName}/{buildNumber}/api/json
   └─ 检查 building 字段和 result 字段
   └─ building=false 时构建完成
3. 返回 BuildCompleteResult (包含产物列表)
```

### 3.5 状态查询服务模块 - `src/services/status-service.ts`

| 属性 | 说明 |
|------|------|
| 路径 | `src/services/status-service.ts` |
| 类名 | `StatusService` |
| 职责 | 查询构建状态,解析 Jenkins API 响应 |
| 依赖 | `HttpClient` |

**核心方法**:

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `getStatus(jobName, buildNumber)` | Job名称, 构建编号或 'last' | `Promise<BuildStatusResult>` | 查询构建状态,返回完整状态信息 |
| `getBuildNumberFromQueue(queueId)` | 队列 ID | `Promise<{buildNumber, jobName} \| null>` | 通过队列 ID 获取构建编号 |
| `parseBuildStatus(data, jobName)` | Jenkins API 原始数据, Job名称 | `BuildStatusResult` | 解析 Jenkins 响应为 SDK 类型 (内部方法) |
| `mapBuildStatus(result, building)` | Jenkins 状态值, building 标志 | `BuildStatus` | 映射 Jenkins 状态到 SDK 状态枚举 (内部方法) |

**状态映射**:

| Jenkins result 值 | building | SDK BuildStatus |
|-------------------|----------|-----------------|
| null/undefined | true | `IN_PROGRESS` |
| 'SUCCESS' | false | `SUCCESS` |
| 'FAILURE' | false | `FAILURE` |
| 'ABORTED' | false | `ABORTED` |
| 'UNSTABLE' | false | `UNSTABLE` |
| 'NOT_BUILT' | false | `NOT_BUILT` |
| 其他 | false | `UNKNOWN` |

### 3.6 下载服务模块 - `src/services/download-service.ts`

| 属性 | 说明 |
|------|------|
| 路径 | `src/services/download-service.ts` |
| 类名 | `DownloadService` |
| 职责 | 处理构建产物的下载,支持单个和批量下载 |
| 依赖 | `HttpClient`, `StatusService`, `fs`, `path` |

**核心方法**:

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `downloadArtifact(jobName, buildNumber, artifactPath, outputDir)` | Job名称, 构建编号, 产物路径, 输出目录 | `Promise<DownloadResult>` | 下载单个产物文件 |
| `downloadAllArtifacts(jobName, buildNumber, outputDir)` | Job名称, 构建编号, 输出目录 | `Promise<DownloadAllResult>` | 下载该构建的所有产物 |
| `ensureDir(dirPath)` | 目录路径 | `void` | 确保目录存在,不存在则递归创建 (内部方法) |

**下载流程**:

```
1. 获取构建信息 (包含产物列表)
   └─ GET /job/{jobName}/{buildNumber}/api/json
2. 验证产物是否存在
3. 构建下载 URL
   └─ /job/{jobName}/{buildNumber}/artifact/{relativePath}
4. 流式下载
   └─ 自动创建目录
   └─ 显示进度日志
5. 返回下载结果 (文件名, 本地路径, 大小, 耗时)
```

### 3.8 错误处理模块 - `src/errors/index.ts`

| 属性 | 说明 |
|------|------|
| 路径 | `src/errors/index.ts` |
| 职责 | 定义所有自定义错误类 |
| 依赖 | 无 |

**错误类继承关系**:

```
Error
 └─ JenkinsError (基础错误, statusCode)
     ├─ AuthenticationError (认证失败, 401/403)
     ├─ JobNotFoundError (Job 不存在, 404)
     ├─ BuildFailedError (构建失败, buildNumber)
     ├─ TimeoutError (操作超时)
     ├─ ArtifactNotFoundError (产物不存在)
     └─ NetworkError (网络错误)
```

**错误类详情**:

| 错误类 | 说明 | 额外属性 |
|--------|------|----------|
| `JenkinsError` | 基础错误类 | `statusCode?: number` |
| `AuthenticationError` | 认证失败 (用户名或 Token 错误) | 无 |
| `JobNotFoundError` | Job 不存在 | 无 |
| `BuildFailedError` | 构建执行失败 | `buildNumber: number` |
| `TimeoutError` | 操作超时 (等待构建、请求超时) | 无 |
| `ArtifactNotFoundError` | 指定的产物不存在 | 无 |
| `NetworkError` | 网络连接问题 | 无 |

## 4. 示例代码说明
