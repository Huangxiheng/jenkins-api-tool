# Jenkins API Tool - 详细设计文档

## 1. 架构设计

### 1.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                    SDK 使用者                           │
└──────────────────────┬──────────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────────┐
│                  JenkinsClient                          │
│  (对外暴露的主入口类)                                     │
│  - build()                                              │
│  - getStatus()                                          │
│  - download()                                           │
│  - downloadAll()                                        │
└─────┬──────────────────────┬────────────────────────────┘
      │                      │
┌─────▼─────┐       ┌───────▼───────┐      ┌──────────────┐
│ Build     │       │ Status        │      │ Download     │
│ Service   │       │ Service       │      │ Service      │
└─────┬─────┘       └───────┬───────┘      └──────┬───────┘
      │                     │                     │
┌─────▼─────────────────────▼─────────────────────▼───────┐
│                    HTTP Client                          │
│  (基于 axios 封装的底层 HTTP 请求)                       │
│  - 认证处理                                             │
│  - 错误处理                                             │
│  - 重试机制                                             │
└──────────────────────┬──────────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────────┐
│                  Jenkins Server                         │
└─────────────────────────────────────────────────────────┘
```

### 1.2 设计原则
- **单一职责**: 每个服务类只负责一个功能域
- **依赖注入**: HTTP Client 作为依赖注入到各服务
- **类型安全**: 完整的 TypeScript 类型定义
- **错误透明**: 自定义错误类型,保留原始错误信息

## 2. 类型定义设计

### 2.1 配置类型

```typescript
// JenkinsClient 配置
interface JenkinsClientConfig {
  url: string;              // Jenkins 服务器地址
  username: string;         // 用户名
  password?: string;        // 密码 (与 apiToken 二选一)
  apiToken?: string;        // API Token (与 password 二选一)
  timeout?: number;         // 请求超时时间(ms), 默认 30000
  retries?: number;         // 重试次数, 默认 0
  retryDelay?: number;      // 重试延迟(ms), 默认 1000
  logLevel?: LogLevel;      // 日志级别, 默认 'info'
}

// 日志级别
type LogLevel = 'debug' | 'info' | 'warn' | 'error' | 'silent';
```

### 2.2 构建参数类型

```typescript
// 构建参数 (键值对形式)
interface BuildParameters {
  [key: string]: string | boolean | FileParameter;
}

// 文件参数
interface FileParameter {
  type: 'file';
  path: string;             // 本地文件路径
}

// 构建选项
interface BuildOptions {
  wait?: boolean;           // 是否等待构建完成, 默认 false
  pollInterval?: number;    // 轮询间隔(ms), 默认 5000
  maxWaitTime?: number;     // 最大等待时间(ms), 默认 600000 (10分钟)
  crumbIssuer?: boolean;    // 是否启用 CSRF 保护, 默认 true
}
```

### 2.3 构建结果类型

```typescript
// 构建触发结果
interface BuildTriggerResult {
  queueId: number;          // 队列 ID
  url: string;              // 构建 URL
  jobName: string;          // Job 名称
}

// 构建完成结果 (同步模式返回)
interface BuildCompleteResult extends BuildTriggerResult {
  buildNumber: number;      // 构建编号
  status: BuildStatus;      // 构建状态
  duration: number;         // 构建耗时(ms)
  artifacts: ArtifactInfo[];// 构建产物列表
}

// 构建状态
type BuildStatus = 
  | 'SUCCESS'               // 成功
  | 'FAILURE'               // 失败
  | 'ABORTED'               // 中止
  | 'UNSTABLE'              // 不稳定
  | 'IN_PROGRESS'           // 进行中
  | 'NOT_BUILT'             // 未构建
  | 'QUEUED'                // 队列中
  | 'UNKNOWN';              // 未知

// 产物信息
interface ArtifactInfo {
  fileName: string;         // 文件名
  relativePath: string;     // 相对路径
  size?: number;            // 文件大小(字节)
}
```

### 2.4 状态查询结果类型

```typescript
interface BuildStatusResult {
  jobName: string;          // Job 名称
  buildNumber: number;      // 构建编号
  status: BuildStatus;      // 构建状态
  displayName: string;      // 显示名称 (如 "#123")
  description?: string;     // 构建描述
  timestamp: number;        // 开始时间戳
  duration: number;         // 持续时间(ms)
  estimatedDuration?: number; // 预估持续时间(ms)
  building: boolean;        // 是否正在构建
  url: string;              // 构建 URL
  consoleUrl: string;       // 控制台日志 URL
  artifacts: ArtifactInfo[];// 构建产物列表
  causes: BuildCause[];     // 触发原因
  parameters: Parameter[];  // 构建参数
}

// 构建触发原因
interface BuildCause {
  shortDescription: string; // 简短描述
  userId?: string;          // 触发用户
  userName?: string;        // 用户名
}

// 参数信息
interface Parameter {
  name: string;
  value: string | boolean;
}
```

### 2.5 下载结果类型

```typescript
interface DownloadResult {
  fileName: string;         // 文件名
  localPath: string;        // 本地保存路径
  size: number;             // 文件大小(字节)
  duration: number;         // 下载耗时(ms)
}

interface DownloadAllResult {
  total: number;            // 总文件数
  success: number;          // 成功数
  failed: number;           // 失败数
  results: DownloadResult[];// 详细结果列表
}
```

## 3. 核心类设计

### 3.1 JenkinsClient (主入口)

```typescript
class JenkinsClient {
  private config: JenkinsClientConfig;
  private httpClient: HttpClient;
  private buildService: BuildService;
  private statusService: StatusService;
  private downloadService: DownloadService;

  constructor(config: JenkinsClientConfig);
  
  // 触发构建
  build(
    jobName: string, 
    params?: BuildParameters, 
    options?: BuildOptions
  ): Promise<BuildTriggerResult | BuildCompleteResult>;
  
  // 查询构建状态
  getStatus(
    jobName: string, 
    buildNumber: number | 'last'
  ): Promise<BuildStatusResult>;
  
  // 下载单个产物
  download(
    jobName: string,
    buildNumber: number,
    artifactPath: string,
    outputDir: string
  ): Promise<DownloadResult>;
  
  // 下载所有产物
  downloadAll(
    jobName: string,
    buildNumber: number,
    outputDir: string
  ): Promise<DownloadAllResult>;
  
  // 获取构建日志
  getConsoleText(
    jobName: string,
    buildNumber: number
  ): Promise<string>;
}
```

### 3.2 HttpClient (HTTP 客户端)

```typescript
class HttpClient {
  private axiosInstance: AxiosInstance;
  private config: JenkinsClientConfig;

  constructor(config: JenkinsClientConfig);
  
  // 初始化认证
  private setupAuth(): void;
  
  // 获取 CSRF crumb (如果启用)
  private fetchCrumb(): Promise<string | null>;
  
  // GET 请求
  get<T>(path: string, params?: Record<string, any>): Promise<T>;
  
  // POST 请求
  post<T>(path: string, data?: any): Promise<T>;
  
  // 下载文件 (流式)
  download(
    path: string, 
    outputPath: string,
    onProgress?: (progress: number) => void
  ): Promise<void>;
  
  // 重试逻辑
  private retryRequest<T>(requestFn: () => Promise<T>): Promise<T>;
}
```

### 3.3 BuildService (构建服务)

```typescript
class BuildService {
  private httpClient: HttpClient;
  private statusService: StatusService;

  constructor(httpClient: HttpClient, statusService: StatusService);
  
  // 触发构建
  async trigger(
    jobName: string,
    params?: BuildParameters,
    options?: BuildOptions
  ): Promise<BuildTriggerResult>;
  
  // 等待构建完成 (轮询)
  async waitForCompletion(
    jobName: string,
    queueId: number,
    options: Required<Pick<BuildOptions, 'pollInterval' | 'maxWaitTime'>>
  ): Promise<BuildCompleteResult>;
  
  // 准备请求参数
  private prepareParams(params: BuildParameters): Promise<any>;
  
  // 处理文件参数
  private handleFileParams(params: BuildParameters): Promise<FormData>;
}
```

### 3.4 StatusService (状态查询服务)

```typescript
class StatusService {
  private httpClient: HttpClient;

  constructor(httpClient: HttpClient);
  
  // 查询构建状态
  async getStatus(
    jobName: string,
    buildNumber: number | 'last'
  ): Promise<BuildStatusResult>;
  
  // 通过队列 ID 获取构建编号
  async getBuildNumberFromQueue(
    queueId: number
  ): Promise<{ buildNumber: number; jobName: string } | null>;
  
  // 解析构建状态
  private parseBuildStatus(
    data: any
  ): BuildStatusResult;
}
```

### 3.5 DownloadService (下载服务)

```typescript
class DownloadService {
```

## 4. 错误处理设计

### 4.1 自定义错误类

```typescript
// 基础错误类
class JenkinsError extends Error {
  name = 'JenkinsError';
  statusCode?: number;
  
  constructor(message: string, statusCode?: number);
}

// 认证错误
class AuthenticationError extends JenkinsError {
  name = 'AuthenticationError';
}

// Job 不存在错误
class JobNotFoundError extends JenkinsError {
  name = 'JobNotFoundError';
}

// 构建失败错误
class BuildFailedError extends JenkinsError {
  name = 'BuildFailedError';
  buildNumber: number;
  
  constructor(message: string, buildNumber: number);
}

// 超时错误
class TimeoutError extends JenkinsError {
  name = 'TimeoutError';
}

// 产物不存在错误
class ArtifactNotFoundError extends JenkinsError {
  name = 'ArtifactNotFoundError';
}

// 网络错误
class NetworkError extends JenkinsError {
  name = 'NetworkError';
}
```

### 4.2 错误处理流程

```
HTTP 请求失败
    │
    ├─ 401/403 → 抛出 AuthenticationError
    ├─ 404     → 抛出 JobNotFoundError / ArtifactNotFoundError
    ├─ 5xx     → 重试 (如果配置了重试)
    │              └─ 重试失败 → 抛出 NetworkError
    ├─ 超时     → 抛出 TimeoutError
    └─ 其他     → 抛出 JenkinsError
```

## 5. 核心流程设计

### 5.1 异步触发构建流程

```
client.build('my-job', { ENV: 'prod' })
    │
    ├─ 1. 构建请求 URL: /job/my-job/buildWithParameters
    │
    ├─ 2. 获取 CSRF crumb (如果启用)
    │      └─ GET /crumbIssuer/api/json
    │
    ├─ 3. 准备请求参数
    │      └─ 转换为 Jenkins 接受的格式
    │
    ├─ 4. 发送 POST 请求
    │      └─ 返回 Queue ID
    │
    └─ 5. 返回 BuildTriggerResult
```

### 5.2 同步触发构建流程 (等待完成)

```
client.build('my-job', { ENV: 'prod' }, { wait: true })
    │
    ├─ 1. 异步触发构建 (同上)
    │
    ├─ 2. 轮询队列状态
    │      └─ GET /queue/item/{queueId}/api/json
    │      └─ 检查 executable 字段
    │
    ├─ 3. 获取构建编号后,轮询构建状态
    │      └─ GET /job/my-job/{buildNumber}/api/json
    │      └─ 检查 building 字段和 result 字段
    │
    ├─ 4. 构建完成
    │      └─ 获取产物列表
    │
    └─ 5. 返回 BuildCompleteResult
```

### 5.3 查询构建状态流程

```
client.getStatus('my-job', 123)
    │
    ├─ 1. 构建请求 URL
    │      └─ /job/my-job/123/api/json
    │      └─ 树参数: tree=number,result,timestamp,duration,...
    │
    ├─ 2. 发送 GET 请求
    │
    ├─ 3. 解析响应数据
    │      └─ 映射到 BuildStatusResult
    │      └─ 计算状态、时间等
    │
    └─ 4. 返回 BuildStatusResult
```

### 5.4 下载构建产物流程

```
client.download('my-job', 123, 'target/app.jar', './dist')
    │
    ├─ 1. 获取构建信息 (获取产物列表)
    │      └─ GET /job/my-job/123/api/json
    │
    ├─ 2. 验证产物是否存在
    │
    ├─ 3. 构建下载 URL
    │      └─ /job/my-job/123/artifact/target/app.jar
    │
    ├─ 4. 流式下载文件
    │      └─ 显示进度
    │
    ├─ 5. 保存到本地
    │      └─ ./dist/app.jar
    │
    └─ 6. 返回 DownloadResult
```

## 11. 安全设计

### 11.1 认证安全
- API Token 不会打印到日志
- .env 文件加入 .gitignore
- 支持 HTTPS 连接

### 11.2 CSRF 保护
- 默认启用 crumbIssuer
- 自动获取和传递 CSRF token
- 支持禁用 (如果 Jenkins 未启用 CSRF)

## 12. 性能优化

### 12.1 HTTP 优化
- 连接复用 (axios keep-alive)
- 请求超时控制
- 重试机制 (指数退避)

### 12.2 下载优化
- 流式下载 (避免大文件内存溢出)
- 并发控制 (下载多个文件时)
- 进度回调

### 12.3 轮询优化
- 可配置轮询间隔
- 最大等待时间限制
- 智能退避策略 (可选)
