## 一、功能概述

### 1.1 功能描述

分支跟踪检查功能用于在自动化构建流程中确保 Git 分支状态的正确性。该功能会自动检查当前分支是否是目标分支,并验证分支的跟踪关系是否正确。如果不满足条件,会自动重建分支以确保代码是最新的,并且跟踪关系正确。

### 1.2 核心需求

1. **检查当前分支**：验证当前分支是否是目标分支 `TARGET_BRANCH`
2. **验证跟踪关系**：检查当前分支是否跟踪正确的远程分支 `origin/TARGET_BRANCH`
3. **自动重建分支**：如果不满足条件,通过以下步骤重建分支：
   - 切换到中间分支 `DEV_TEST_BRANCH`
   - 删除本地的目标分支
   - 从远程仓库同名的目标分支新建本地目标分支

### 1.3 应用场景

- 自动化构建流程前确保分支状态正确
- 多人协作项目中保证代码同步
- CI/CD 流水线中的分支管理
- 确保本地分支与远程分支的跟踪关系正确

## 二、实现原理

### 2.1 核心逻辑流程图

```
开始
  ↓
检查当前分支名称
  ↓
当前分支 == TARGET_BRANCH？
  ├─ 是 → 检查跟踪分支
  │        ↓
  │     跟踪分支 == origin/TARGET_BRANCH？
  │        ├─ 是 → 完成(无需重建)
  │        └─ 否 → 重建分支
  │
  └─ 否 → 重建分支
           ↓
        步骤1: 切换到 DEV_TEST_BRANCH
           ↓
        步骤2: 删除本地 TARGET_BRANCH
           ↓
        步骤3: 从远程新建本地 TARGET_BRANCH
           ↓
        步骤4: 验证跟踪关系
           ↓
        完成
```

### 2.2 关键技术点

#### 2.2.1 获取跟踪分支

使用 Git 命令获取当前分支的上游分支：

```bash
git rev-parse --abbrev-ref --symbolic-full-name @{u}
```

**命令说明**：
- `@{u}` 表示当前分支的上游分支(upstream)
- `--abbrev-ref` 使用简短引用名称
- `--symbolic-full-name` 显示完整的符号名称

**返回值**：
- 成功时返回跟踪分支名称,例如 `origin/Feature_20260130_chongQingWenLvWei`
- 如果没有设置跟踪分支,命令会报错,函数返回空字符串

#### 2.2.2 分支重建流程

分支重建的核心步骤(精简为3个核心步骤+验证):

1. **切换到中间分支**：确保不在目标分支上才能删除它
   ```bash
   git checkout DEV_TEST_BRANCH
   ```

2. **删除本地目标分支**：清除旧的本地分支
   ```bash
   git branch -D TARGET_BRANCH
   ```

3. **从远程新建本地分支**：一条命令完成创建+跟踪+切换
   ```bash
   git checkout -b TARGET_BRANCH origin/TARGET_BRANCH
   ```
   该命令会：
   - 创建本地分支 `TARGET_BRANCH`
   - 自动跟踪远程分支 `origin/TARGET_BRANCH`
   - 自动切换到新创建的分支

4. **验证跟踪关系**：确保操作成功
   ```bash
   git rev-parse --abbrev-ref --symbolic-full-name @{u}
   ```

## 三、详细实现

### 3.1 配置常量

```typescript
// 目标分支名称
const TARGET_BRANCH = 'Feature_20260130_chongQingWenLvWei';

// 中间切换分支(用于重置跟踪关系)
const DEV_TEST_BRANCH = 'dev_test';
```

**说明**：
- `TARGET_BRANCH`：最终要切换到的目标分支
- `DEV_TEST_BRANCH`：中间过渡分支,用于在删除目标分支时提供安全的切换位置

### 3.2 核心函数实现

#### 3.2.1 获取跟踪分支函数

```typescript
async function getUpstreamBranch(): Promise<string> {
  try {
    // 使用 git rev-parse 获取当前分支的跟踪分支
    const upstream = (await runGitBashCommand(
      `cd ${UNIAPP_PROJECT_PATH} && git rev-parse --abbrev-ref --symbolic-full-name @{u}`
    )).trim();

    return upstream || '';
  } catch (error) {
    // 如果没有设置跟踪分支,返回空字符串
    return '';
  }
}
```

**特点**：
- 异步函数,使用 `async/await` 处理 Promise
- 错误捕获机制,如果没有跟踪分支返回空字符串
- 自动去除输出结果的首尾空格

#### 3.2.2 分支检查与重建函数

```typescript
async function checkAndSwitchBranch(): Promise<void> {
  // 1. 获取当前分支
  const currentBranch = (await runGitBashCommand(
    `cd ${UNIAPP_PROJECT_PATH} && git branch --show-current`
  )).trim();

  // 2. 判断是否需要重建
  let needRebuild = false;
  const expectedUpstream = `origin/${TARGET_BRANCH}`;

  if (currentBranch === TARGET_BRANCH) {
    // 已经在目标分支,检查跟踪关系
    const upstream = await getUpstreamBranch();
    if (upstream !== expectedUpstream) {
      needRebuild = true;
    }
  } else {
    // 不在目标分支,需要重建
    needRebuild = true;
  }

  // 3. 执行重建流程
  if (needRebuild) {
    // 步骤1：切换到中间分支
    await runGitBashCommand(`cd ${UNIAPP_PROJECT_PATH} && git checkout ${DEV_TEST_BRANCH}`);

    // 步骤2：删除本地目标分支
    try {
      await runGitBashCommand(`cd ${UNIAPP_PROJECT_PATH} && git branch -D ${TARGET_BRANCH}`);
    } catch (error) {
      // 分支不存在时忽略错误
    }

    // 步骤3：从远程同名的目标分支新建本地目标分支
    // 一条命令完成：创建分支 + 建立跟踪关系 + 切换到分支
    await runGitBashCommand(`cd ${UNIAPP_PROJECT_PATH} && git checkout -b ${TARGET_BRANCH} origin/${TARGET_BRANCH}`);

    // 步骤4：验证跟踪关系
    const newUpstream = await getUpstreamBranch();
    if (newUpstream !== expectedUpstream) {
      throw new Error('分支重建失败,跟踪关系不正确');
    }
  }
}
```

**关键改进**：
- 步骤3使用 `git checkout -b` 命令,一条命令完成创建+跟踪+切换
- 流程更简洁,从原来的5步减少到3个核心步骤+验证
- 自动建立正确的跟踪关系,无需额外操作

## 五、错误处理

### 5.1 错误类型

| 错误场景 | 处理方式 | 影响 |
|---------|---------|------|
| 中间分支不存在 | 抛出异常,终止流程 | 流程中断 |
| 远程分支不存在 | Git 命令失败,抛出异常 | 流程中断 |
| 网络问题导致 checkout 失败 | Git 命令失败,抛出异常 | 流程中断 |
| 本地分支不存在(删除时) | 捕获异常,忽略错误 | 继续执行 |
| 跟踪关系验证失败 | 抛出异常,终止流程 | 流程中断 |

### 5.2 错误处理示例

```typescript
try {
  await runGitBashCommand(`cd ${UNIAPP_PROJECT_PATH} && git branch -D ${TARGET_BRANCH}`);
  console.log(`[成功] 已删除本地 ${TARGET_BRANCH} 分支`);
} catch (error) {
  // 如果分支不存在,忽略错误
  console.log(`[提示] 本地 ${TARGET_BRANCH} 分支不存在,跳过删除`);
}
```

## 六、关键设计决策

### 6.1 为什么需要中间分支？

**问题**：无法删除当前所在的分支

**解决方案**：
1. 先切换到中间分支 `DEV_TEST_BRANCH`
2. 然后删除目标分支
3. 最后从远程重新新建本地分支

**优点**：
- 安全性高,不会误删其他分支
- 保证删除操作的可行性
- 提供回退的机会

### 6.2 为什么使用 `git checkout -b` 而不是分步操作？

**对比**：

| 方法 | 命令数量 | 步骤复杂度 | 跟踪关系 |
|------|---------|-----------|---------|
| **推荐方法** | 1条命令 | 简单 | ✅ 自动建立 |
| 分步方法 | 2条命令 | 复杂 | ✅ 需额外配置 |
| fetch方法 | 2条命令 | 中等 | ⚠️ 可能不自动跟踪 |

**推荐方法**：
```bash
git checkout -b TARGET_BRANCH origin/TARGET_BRANCH
```

**优势**：
- ✅ 一条命令完成三个操作：创建分支、建立跟踪、切换分支
- ✅ 自动建立正确的跟踪关系 `origin/TARGET_BRANCH`
- ✅ 代码更简洁,维护成本更低
- ✅ 减少出错概率

**分步方法**(不推荐)：
```bash
# 需要两步
git fetch origin TARGET_BRANCH:TARGET_BRANCH
git checkout TARGET_BRANCH
# 可能还需要额外设置跟踪关系
git branch --set-upstream-to=origin/TARGET_BRANCH TARGET_BRANCH
```

**选择理由**：
- `git checkout -b` 是 Git 官方推荐的标准方式
- 自动处理跟踪关系,无需额外配置
- 符合"从远程仓库同名的目标分支新建本地目标分支"的需求
- 更加符合 Git 的最佳实践

### 6.3 为什么需要验证步骤？

**必要性**：
1. 确保操作成功
2. 及时发现问题
3. 提供明确的反馈

**验证内容**：
- 跟踪分支是否正确建立
- 本地分支是否正确创建
- 整体状态是否符合预期

## 八、性能考虑

### 8.1 执行时间

| 操作 | 预估时间 | 影响因素 |
|------|---------|---------|
| 检查当前分支 | < 1s | Git 命令执行速度 |
| 获取跟踪分支 | < 1s | Git 命令执行速度 |
| 切换分支 | 1-3s | 文件变更数量 |
| 删除分支 | < 1s | Git 操作 |
| 从远程新建分支 | 3-10s | 网络速度、分支大小 |
| 验证跟踪关系 | < 1s | Git 命令执行速度 |

**总耗时**：
- 最佳情况(无需重建)：< 2 秒
- 最差情况(需要重建)：5-15 秒

### 8.2 优化建议

1. **并行检查**：可以并行执行多个检查命令
2. **缓存结果**：避免重复检查
3. **提前验证**：在流程开始前验证必要条件

## 九、安全性考虑

### 9.1 潜在风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 未提交的更改丢失 | 高 | 建议先 stash 或 commit |
| 中间分支不存在 | 中 | 提前验证 |
| 远程分支不存在 | 高 | 提前验证并提供错误信息 |
| 网络故障 | 中 | 重试机制和超时处理 |

### 9.2 安全措施

```typescript
// 建议：添加安全检查
async function safeCheckAndSwitchBranch(): Promise<void> {
  // 1. 检查是否有未提交的更改
  const status = await runGitBashCommand(`cd ${UNIAPP_PROJECT_PATH} && git status --porcelain`);
  if (status.trim()) {
    throw new Error('存在未提交的更改,请先提交或暂存');
  }

  // 2. 检查中间分支是否存在
  const branches = await runGitBashCommand(`cd ${UNIAPP_PROJECT_PATH} && git branch --list ${DEV_TEST_BRANCH}`);
  if (!branches.includes(DEV_TEST_BRANCH)) {
    throw new Error(`中间分支 ${DEV_TEST_BRANCH} 不存在`);
  }

  // 3. 执行分支检查
  await checkAndSwitchBranch();
}
```

### B. Git checkout -b 命令详解

**命令格式**：
```bash
git checkout -b <new-branch-name> <start-point>
```

**参数说明**：
- `-b`：创建新分支
- `<new-branch-name>`：新分支的名称
- `<start-point>`：新分支的起始点,可以是提交、分支名或远程分支

**示例**：
```bash
# 从远程分支创建本地分支并建立跟踪
git checkout -b Feature_20260130_chongQingWenLvWei origin/Feature_20260130_chongQingWenLvWei
```

**执行结果**：
1. 创建本地分支 `Feature_20260130_chongQingWenLvWei`
2. 设置跟踪关系为 `origin/Feature_20260130_chongQingWenLvWei`
3. 自动切换到新创建的分支
4. 新分支的内容与远程分支一致
